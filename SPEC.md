# SPEC: Family Tree — Генеалогическое веб-приложение

## 1. Назначение

Веб-приложение для создания, визуализации и управления родовым (генеалогическим) деревом семьи. Позволяет добавлять родственников, устанавливать семейные связи, загружать фото/видео, хранить биографии и экспортировать дерево в SVG/JSON.

---

## 2. Технологический стек

| Категория | Технология |
|-----------|-----------|
| **Язык фронтенда** | JavaScript (Vanilla ES6) |
| **Язык бэкенда** | Python 3.13 (стандартная библиотека) |
| **Разметка** | HTML5 |
| **Стилизация** | CSS3 (custom properties, glassmorphism, адаптив) |
| **Шрифты** | Manrope + Cormorant Garamond (Google Fonts) |
| **Хранилище** | `localStorage` (данные деревьев), файловая система `media/` (медиафайлы) |
| **Сервер** | `ThreadingHTTPServer`, порт **127.0.0.1:8000** |
| **Формат данных** | JSON |

### 3.2 Клиент-серверное взаимодействие

```
┌─────────────┐         HTTP GET / → index.html        ┌─────────────┐
│             │ ◄───────────────────────────────────── │             │
│  Браузер    │                                         │  Сервер    │
│  (HTML/CSS  │         HTTP POST /api/upload          │  Python     │
│   /JS)      │ ─────────────────────────────────────► │  :8000      │
│             │         multipart/form-data             │             │
│             │ ◄───────────────────────────────────── │             │
│             │         {"path": "/media/photo.jpg"}    │             │
└─────────────┘                                         └─────────────┘
```

- Сервер раздаёт статику и обрабатывает загрузку медиа.
- Все данные о родственниках хранятся в `localStorage` на клиенте.
- Медиафайлы сохраняются на сервере в `media/` с UUID-суффиксом.

## 5. API endpoints

### `POST /api/upload`

Загрузка медиафайла на сервер.

**Request:** `multipart/form-data` с полем `file`

**Response (200):**
```json
{ "path": "/media/myphoto-a1b2c3d4.jpg" }
```

**Response (400):**
```json
{ "error": "Expected multipart/form-data" }
```

**Response (500):**
```json
{ "error": "Upload failed: ..." }
```

Файлы сохраняются в `media/` с нормализованным именем + UUID суффиксом.

## 6. Модели данных

### 6.1 `Member` (родственник)

```typescript
interface Member {
  id: string;                    // UUID
  name: string;                  // "Иван Орлов"
  relationType: RelationType;    // "ancestor" | "parent" | "child" | "partner" | "relative"
  gender: string;                // "female" | "male" | "other" | ""
  maidenName: string;            // Девичья фамилия
  years: string;                 // "1941 - 2016"
  birthDate: string;             // "1941-08-19"
  deathDate: string;             // "2016-01-28"
  location: string;              // "Полтава"
  birthPlace: string;            // "Полтава, Украина"
  familyBranch: string;          // "Орловы"
  kinship: string;               // "Дедушка"
  generation: number;            // 1..8
  motherId: string;              // ID матери или ""
  fatherId: string;              // ID отца или ""
  partnerId: string;             // ID партнёра или ""
  childrenIds: string[];         // Массив ID детей
  bio: string;                   // Биография
  highlights: string[];          // Массив фактов
  photo: string;                 // Путь к фото
  video: string;                 // Путь к видео
}
```

### 6.2 `Tree` (дерево)

```typescript
interface Tree {
  id: string;                    // UUID
  name: string;                  // "Семья Орловых"
  members: Member[];             // Члены дерева
  nodePositions: Record<string, { x: number }>; // Позиции узлов (горизонталь)
  selectedId: string;            // ID выбранного члена
}
```

### 6.3 `Forest` (набор деревьев)

Хранится в `localStorage` под ключом `"family-tree-forest"` как `Tree[]`.

## 7. Алгоритмы

### 7.1 Авторасстановка (getDefaultTreeLayout)

1. Группировка членов по поколению (`generation`)
2. Внутри поколения — группировка по родительской паре или партнёрской паре
3. Приоритизация выбранного элемента (центрирование)
4. Расчёт X-координат с учётом ширины карточки (82px), зазоров (10px) и отступов (44px)
5. Y-координаты: фиксированный шаг 190px между поколениями

### 7.2 Пересчёт поколений (recalculateGenerations)

Итеративный алгоритм:
1. Для каждого члена: найти max(generation) среди родителей
2. Если родитель найден — generation = родительское + 1
3. Также синхронизировать поколение партнёра
4. Повторять до стабилизации (максимум кол-во членов + 3 итераций)

### 7.3 Отрисовка связей (drawTreeLinks)

1. Сбор видимых узлов
2. Построение карты семей: motherId + fatherId → children[]
3. Для каждой семьи: родительский bus (горизонтальная линия) → вертикальные линии к детям
4. Для партнёрских пар: горизонтальная линия между партнёрами
5. Используются `getNodeMetrics()` для вычисления координат

## 11. Критерии приёмки (Acceptance Criteria)

1. **AC1:** Пользователь может создать новое дерево, добавить родственника с именем и ролью, и увидеть его карточку в дереве.
2. **AC2:** Пользователь может кликнуть на карточку в дереве и увидеть детальную информацию справа.
3. **AC3:** Пользователь может перетаскивать карточки в дереве; положение сохраняется после перезагрузки.
4. **AC4:** Пользователь может экспортировать дерево в SVG и JSON, затем импортировать JSON обратно.
5. **AC5:** Поиск фильтрует карточки по имени; фильтр по роли показывает только выбранную категорию.
6. **AC6:** Пользователь может загрузить фото родственника; фото отображается в карточке деталей.
7. **AC7:** При удалении родственника все связи (как родитель, партнёр, ребёнок) очищаются у других членов.
8. **AC8:** Поколения автоматически пересчитываются при добавлении/редактировании родителей и детей.
9. **AC9:** Интерфейс корректно отображается на мобильных устройствах (адаптивная сетка).
10. **AC10:** При загрузке страницы восстанавливается последнее активное дерево и выбор.

## 12. Известные ограничения

- Нет серверной базы данных (только `localStorage`)
- Нет авторизации / многопользовательского режима
- Нет поддержки биологических родителей и приёмных (один набор motherId/fatherId)
- Автосохранение только при переключении дерева или изменении данных (не в реальном времени при drag)
- SVG-экспорт не включает фоновые изображения
